Friends

A friend is anyone who the player can interact with and cannot attack. They can have complex dialogue trees and can even change the dialogue after receiving gifts.

Descriptions

As with items, friends have two descriptions, one that will be used when talking to the friend, and one that is used to describe them being in a room.

Dialogue

A Friend's Dialogue is comprised of lines of speech that the player can respond to. If you only want a single line of dialogue this is achievable, just don't add any responses.

If you want a complex tree of dialogue then you can select the dialogue option you want to add responses to and then add the response. A response needs a "talking point" which is the line of dialogue the player is selecting to say in response to the friend's speech. Then the friend has a response to that.

Dialogue can include the player's name to make it feel more real. Include [PLAYER_NAME] (all caps with the surrounding square brackets) and when a player plays the game it will automatically replace it with their name. For example the text "Hello, [PLAYER_NAME], how are you?" would display as "Hello, John, how are you?" for a player with the name John.

Name Changes

It's possible to add a change to a friends name, accepted names, description and in room description based on a specific line of dialogue being shown. This way you can hide a friends true name and description until they learn it through interacting with the friend.

Received Gifts

At any point in the dialogue tree you can add a list of items that are given to the player by the friend when that line of dialogue is printed out.

Accepted Gifts

The final thing that is available with friends is Accepted Gifts. This is a set of items that can be given by the player to the friend. These items have the ability to hide certain sections of the dialogue and reveal others. This way you can set up quests that require fetching an item for a reward.

When creating the dialogue, you can set it to start as hidden. This way it won't be available to the player at first, until they have given the item that is set to reveal that dialogue. The earliest point in the dialogue that needs to be hidden or reveal is the only part that needs to be selected at this point, all the responses to it will be hidden or reveal implicitly.

You can take this to the extreme by replacing the entire dialogue tree by creating a new starting point entirely, though only one starting point can start as not hidden. Creating a new non-hidden starting point will automatically set any other starting points to hidden.

Shopkeepers

Any friend in your game can be turned into a shopkeeper by adding an item to their Sold items on their details page.

You'll need to first create a currency in the Currencies tab and after that by selecting the item, the price, and the currency you want for it, this will now provide the option to the player to "buy items" when talking to the friend.

Example

A jailor stands guard over the prison but you need access to a cell. In order to get in the cell the guard can give you the key but he wants a bottle of beer in return. So you add some dialogue to the guard asking the player what they want, and then inform them they are a bit thirsty. Another starting point of the dialogue is added - hidden at first - to thank the player for the beer and give the key to the player. You add an accepted gift of a bottle of beer item that hides the thirsty dialogue and reveals the thankful dialogue.
